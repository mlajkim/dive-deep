## Prerequisites

Here is the prerequisite setup for the following examples:

```sh
test_name=redirection_test
tmp_dir=$(date +%y%m%d_%H%M%S_$test_name)
mkdir -p ~/test_dive/$tmp_dir
cd ~/test_dive/$tmp_dir

echo "Target Acquired" > existing_file
ls -al existing_file ghost_file
# ls: ghost_file: No such file or directory
# -rw-r--r--  1 ajk  staff  16 Dec 15 12:27 existing_file
```


## 1> or >: Redirects only stdout

> [!NOTE]
> Note that `1>` and `>` are equivalent.

```sh
ls -al existing_file ghost_file 1> success_log_only.log 2> /dev/null
cat success_log_only.log

# -rw-r--r--  1 ajk  staff  16 Dec 15 12:27 existing_file
```

## 2>: Redirects only stderr

> [!NOTE]
> Note that the special directory `/dev/null` is used to discard unwanted successful output for the not-confusing error log.

```sh
ls -al existing_file ghost_file > /dev/null 2> error_log_only.log
cat error_log_only.log

# ls: ghost_file: No such file or directory
```

## &> Redirects both stdout and stderr

```sh
ls -al existing_file ghost_file &> combined_log.log
cat combined_log.log

# ls: ghost_file: No such file or directory
# -rw-r--r--  1 ajk  staff  34 Dec 15 12:37 existing_file
```

## >>: Appends output to a file

```sh
echo "New Line" >> existing_file
cat existing_file

# Target Acquired
# New Line
```

## <: Input Redirection

> [!NOTE]
> Note that "Input Redirection" is the official name of the `<` operator.

Note that `New Line` is now the first line of the sorted output:

```sh
sort < existing_file

# New Line
# Target Acquired
```

## <<: Here Document

> [!NOTE]
> Note that "Here Document" is the official name of the `<<` operator.

```sh
bc << EOF
scale=2
10/3
EOF
# 3.33
```

## <<<: Here string

You can also use `<<<` to pass a string as input to a command:

```sh
bc <<< "scale=2; 10/3"
# 3.33
```
